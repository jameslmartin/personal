Security concerns in automotive systems

18 Feb. 2015

On February 18th, I gave a presentation about security vulnerabilities in modern automotive vehicles. The presentation was for a graduate level course led by Jim Anderson on autonomous automotive vehicles. I performed some first-pass research on vulnerabilities that modern cars face, today. Synthesizing the information, I created a presentation that summarized my findings. Click here to view the full presentation and download the PDF.

Attack surfaces studied:

  • OBD-II port (federally mandated in all US vehicles)
  • Bluetooth channels
  • Addressable channels through technologies such as OnStar, etc.

Attacks studied:

  • Stuxnet-esque worm infecting dealership network / physical attack on dealership, malware installed on OBD-II PassThru devices through shell injection, binaries installed on vehicle ECU
  • Bluetooth brute force pairing, infect vehicle ECU
  • Reverse engineered aqLink protocol, input validation bugs of Gateway program leads to easy buffer overflow after buggy authentication. Malware installed via addressable channel over 3G or over phone

Recommendations:

  • Sanitize inputs.
  • Avoid use of unsafe library functions such as strncopy.
  • Hieriarchical CAN buses by critical/non-critical components instead of bandwidth.
  • More code transparency, hire security contractors.

Other concerns:

  • Privacy concerns with data collection.
    • Audio being recorded in car
    • Driving history
    • Third party acquisition of mentioned information
  • Certification process overlooks software that ships with vehicle.
  • Lack of response from manufacturers.
